# Conso 任务奖励系统测试用例

## 1. 测试目的

本文件用于记录 Conso 任务奖励系统的手动测试用例。

每次修改代码后，需要根据本文件检查核心功能是否正常，避免出现以下问题：

* 新功能能跑，但旧功能被改坏
* 终端显示成功，但 JSON 数据没有正确保存
* 程序报错但日志没有记录
* 异常输入没有被正确拦截
* 奖励、任务状态等关键数据出现错误

---

## 2. 测试观察位置

每次测试时，至少观察三个地方：

### 2.1 终端输出

检查用户看到的提示是否正确。

例如：

* 是否提示“任务新增成功”
* 是否提示“输入无效”
* 是否提示“功能执行出错，请检查日志 app.log”

### 2.2 task.json

检查任务数据是否真的发生变化。

例如：

* 新增任务后，是否多了一条任务
* 完成任务后，`task_status` 是否变成 `已完成`
* 领取奖励后，`reward_status` 是否变成 `已领取`
* 删除任务后，任务是否真的从 JSON 中删除

### 2.3 app.log

检查日志是否记录了关键行为。

例如：

* 系统是否启动
* 用户选择了哪个菜单
* 数据是否读取成功
* 数据是否保存成功
* 输入无效时是否有 WARNING
* 功能报错时是否有 ERROR 和 traceback

---

## 3. 每次改代码后的基本测试流程

每次修改代码后，按照以下顺序测试：

1. 保存所有修改过的文件
2. 重新运行程序
3. 测试正常启动
4. 测试正常退出
5. 测试无效菜单输入
6. 测试本次修改影响到的功能
7. 查看 `task.json`
8. 查看 `app.log`
9. 确认没有异常后，再提交 Git
